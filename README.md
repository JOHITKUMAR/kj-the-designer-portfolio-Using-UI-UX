# KJ The Designer — Portfolio Website

A personal portfolio website for **Johit Kumar**, a UI/UX & Product Designer and CSE student based in Jaipur, Rajasthan. The site was designed in **Figma** and built as a fully responsive static website to showcase design work, services, and case studies to potential clients and collaborators.

🔗 **Fignma Project Site Link :**  https://www.figma.com/design/Zp7Adlil7J5jRJ2zXtNqQI/Untitled?node-id=107-92&t=zylAKBpKEKO3Du0S-0

## 📖 About the Project

This repository contains the source code for Johit Kumar's professional design portfolio, branded as **"KJ The Designer."** The goal of the site is simple: give visitors a clear, fast, and visually engaging way to understand who Johit is, what he does, and how he approaches design work — without unnecessary clutter or distraction.

The portfolio reflects Johit's identity as a **UI/UX Designer and Computer Science Engineering (CSE) student** who is passionate about creating clean, intuitive, and engaging digital experiences. The design philosophy behind the site is centered on usability, clarity, and a user-generated design approach — meaning every section was built with the end user's needs and behavior in mind, rather than purely aesthetic decisions.

The site itself acts as a live demonstration of Johit's skill set. Since the entire layout, spacing, color system, and interactive flow were designed in Figma before any code was written, the portfolio is also a showcase of how a structured Figma-first workflow translates into a polished, production-ready website.

## ✨ Features

The portfolio is organized into clear, purposeful sections, each designed to guide a visitor naturally from introduction to contact:

- **Hero Section** — A welcoming introduction with Johit's name, role, and location (Jaipur, India), along with a scroll-down prompt to guide visitors into the rest of the page.
- **Biography Section** — A short, focused bio describing Johit as a UI/UX Designer and CSE student, alongside quick-glance stats such as years of experience and completed projects.
- **About Section** — A deeper introduction explaining Johit's design philosophy: transforming ideas into intuitive, visually appealing, and user-generated digital experiences.
- **Work Progress Timeline** — A visual breakdown of Johit's design process, structured as:
  **Research → Wireframe → UI Design → Prototyping → User Testing → Final Design Delivery.**
  This section highlights how each project moves from concept to a developer-ready final product.
- **Services Section** — An overview of the core services offered:
  - User Experience (UX) Design
  - User Interface (UI) Design
  - Usability Testing
  - Delivery of final, developer-ready assets and specifications
- **Recent Case Studies** — Highlighted project work, including:
  - **Landing Page Design** — A modern, responsive landing page focused on user engagement, clear navigation, and conversion optimization.
  - **E-Commerce App Design** — A user-friendly mobile app design with intuitive navigation and a seamless shopping experience.
- **Client Feedback Section** — Testimonials from past clients reflecting on the quality and impact of the design work delivered.
- **Call-to-Action Sections** — Multiple "Let's Work Together" prompts encouraging visitors to request a quote or start a project conversation.
- **Contact Section** — Direct contact details so potential clients or collaborators can easily reach out.
- **Fully Responsive Layout** — The site adapts cleanly across desktop, tablet, and mobile screen sizes.

## 🛠️ Designed With

- **Figma** — UI/UX design and prototyping

### Figma Features Used

This project was designed entirely in Figma, with the following features used to build a polished, consistent, and developer-ready design:

- **Auto Layout** — Used to maintain consistent spacing, padding, and alignment across every section, so the layout stays balanced even when content changes.
- **Components & Variants** — Buttons, cards, icons, and other repeated UI elements were built as reusable components with variants, ensuring visual consistency throughout the entire portfolio.
- **Prototyping / Interactive Flows** — A clickable prototype was created to simulate real navigation and user flow, allowing the overall experience to be tested and refined before any development work began.
- **Responsive / Constraints** — Layout constraints were applied so that elements resize and reposition predictably across different screen sizes, supporting the site's responsive behavior.
- **Design System** — A consistent design system — including a defined color palette, typography styles, and spacing rules — was maintained across every screen to keep the portfolio cohesive and professional.
